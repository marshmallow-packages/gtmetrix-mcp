Create .github/workflows/ci.yml with this exact content (from 05-RESEARCH.md, verified against astral-sh/setup-uv docs):

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.11", "3.12", "3.13"]
    steps:
      - uses: actions/checkout@v6
      - uses: astral-sh/setup-uv@v7
        with:
          python-version: ${{ matrix.python-version }}
          enable-cache: true
      - name: Install dependencies
        run: uv sync --locked --dev
      - name: Lint
        run: uv run ruff check .
      - name: Test
        run: uv run pytest tests/ -v
```

Key details:
- Python versions MUST be quoted strings ("3.11" not 3.11) to avoid YAML float parsing
- --locked flag ensures CI fails if lockfile is out of date
- enable-cache: true for faster subsequent runs
- Single job is fine for this project size (no need for separate lint/test jobs)
- Triggers: pushes to main + PRs targeting main only (per user decision)
