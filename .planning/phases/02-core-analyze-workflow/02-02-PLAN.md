  <action>
    Create tools/analyze.py following the exact pattern from tools/account.py:

    1. Constants at module level: POLL_INTERVAL = 3, DEFAULT_TIMEOUT = 300

    2. _analyze_impl(client, url: str) -> dict:
       - try block wrapping the entire flow
       - Start test: test = await client.start_test(url); extract test_id = test["id"]; if no id, return error dict
       - Poll loop using time.monotonic() deadline:
         - result = await client.get_test(test_id)
         - If result["type"] == "report": redirect was followed, use result as report, report_id = result["id"], break
         - If result.get("state") == "completed": report_id = test_id, report = await client.get_report(report_id), break
         - If result.get("state") == "error": return error dict immediately with result.get("error") message
         - Else: await asyncio.sleep(POLL_INTERVAL), continue
       - while/else: if loop completes without break, return timeout error dict
       - Fetch sub-resources: lighthouse_json = await client.get_resource(report_id, "lighthouse"), har_json = await client.get_resource(report_id, "har")
       - Parse: import from client.parsers; call extract_vitals(report), filter_failing_audits(lighthouse_json), extract_top_resources(har_json)
       - Return combined dict: {url, test_id, report_id, **vitals, "failing_audits": audits, "top_resources": resources}
       - except httpx.HTTPStatusError: handle 402 (credits) and 429 (concurrent limit) specifically, generic HTTP error for others
       - except Exception: generic error with detail

    3. register(mcp) -> None:
       - Import Context from mcp.server.fastmcp
       - Define @mcp.tool() async def gtmetrix_analyze(url: str, ctx: Context) -> dict with descriptive docstring
       - Extract client from ctx.request_context.lifespan_context["client"]
       - Return await _analyze_impl(client, url)

    Write tests in tests/test_analyze.py:
    - Use AsyncMock for client, patching asyncio.sleep to be a no-op coroutine (avoids real delays)
    - For timeout test, patch time.monotonic to return values that simulate exceeding the deadline

    - Use the MOCK_REPORT_RESPONSE, MOCK_LIGHTHOUSE_RESPONSE, MOCK_HAR_RESPONSE fixtures from conftest.py for the happy path
